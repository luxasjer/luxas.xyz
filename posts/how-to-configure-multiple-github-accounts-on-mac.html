如何在一台 Mac 上配置多个 GitHub 账号 ​

技术教程

问题 ​

有两个 GitHub 账号（一个是工作号 physicsay，一个是私人号 luxasjer），如何在一台 Mac 电脑上配置，且互不影响？

TIPS

你可以直接抄答案，完成配置。

解决方法 ​

用两个邮箱注册了 GitHub 账号，分别为 physicsay(工作账号)、luxasjer(私人账号)。

1. 创建 SSH key ​

(1) 打开终端(Terminal)
(2) 粘贴以下文本，将示例中使用的电子邮件替换为 GitHub 电子邮件地址 A 。

bash
ssh-keygen -t ed25519 -C "physicsay@example.com"
1

终端中会出现一些出现的提示，例如

Generating public/private ed25519 key pair.
....
1
2

直接按回车跳过即可。

(3) 找到 SSH 文件所在文件夹，修改文件名称。

在终端依次输入

bash
cd ~/.ssh
open .
1
2

打开一个名为 .ssh 的文件夹，里面有刚刚生成的 id_ed25519.pub 和 id_ed25519 两个文件。将文件名改成 physicsay_private_key.pub 和 physicsay_private_key(这是用 A 邮箱生成的 SSH key)。

(4) 重复上述 (1)-(3) 过程，生成 B 邮箱的 SSH key，命名为 luxasjer_private_key.pub 和 luxasjer_private_key。

2. 将 SSH 密钥添加到 ssh-agent ​

(1) 启动 ssh-agent

在终端输入

bash
eval "$(ssh-agent -s)"
1

(2) 配置 config 文件

在终端输入

bash
touch ~/.ssh/config
1

即创建了一个名为 config 的文件，用记事本或编辑器打开，输入以下内容：

# Default github account: luxasjer
Host github.com
   HostName github.com
   IdentityFile ~/.ssh/luxasjer_private_key
   IdentitiesOnly yes
   
# Other github account: physicsay
Host github-physicsay
   HostName github.com
   IdentityFile ~/.ssh/physicsay_private_key
   IdentitiesOnly yes
1

注意：

  1. 这里将 luxasjer 的 GitHub 账号设置为默认；
  2. Host 中的 github.com 和 github-physicsay 这个是两个账号的别称 (后面会用到)
  3. HostName 是 github.com，是因为我们配置的是 github 账号。如果你用的是 gist，那么得修改。

(3) 添加 SSH 账号到 ssh-agent

终端依次输入

bash
ssh-add ~/.ssh/github.com
ssh-add ~/.ssh/github-physicsay
1
2

3. 配置 Github 账号 ​

(1) 复制本地公钥

还记得上面生成了两个文件，luxasjer_private_key.pub 和 luxasjer_private_key，其中luxasjer_private_key.pub 是公钥，在终端输入

bash
cat luxasjer_private_key.pub
1

终端会打印出一长串字符，将其全部复制，备用。

(2) 将公钥添加到 GitHub 中

打开 GitHub 官网，右上角 账号 - Settings - SSH and GPG keys - New SSH key，将上面复制的公钥添加进去并保存。

(3) 配置多个 Github 账号

打开另一个 GitHub 官网页面，按照上述 (2) 步骤，添加公钥并保存。

4. 连通测试 ​

终端依次输入：

bash
ssh-keyscan github.com >> ~/.ssh/known_hosts
ssh -T git@github.com
ssh -T git@github-physicsay
1
2
3

如果一切顺利，终端会打印以下内容:

Hi luxasjer! You've successfully authenticated, but GitHub does not provide shell access.
1
Hi physicsay! You've successfully authenticated, but GitHub does not provide shell access.
1

注意：终端会显示 GitHub 的名称。

5. 重要内容 ​

至此，已经将 GitHub 账号与本地 SSH 配置完成，需要记住的是：

git@github.com:org/project.git
git@github-physicsay:org/project.git
1
2

需要修改 org 为 GitHub 名称，project 为仓库名。

个人账号修改为 git@github.com:luxasjer/blog.git ，
公司账号修改为 git@github-physicsay:physicsay/blog.git

(a) 克隆仓库，你需要做：

bash
git clone git@github-physicsay:physicsay/blog.git /path/to/project1
cd /path/to/project1
git config user.email "physicsay@example.com"
git config user.name  "Physicsay"

cd /path/to/project2
git remote set-url origin git@github-physicsay:physicsay/blog.git
git config user.email "physicsay@example.com"
git config user.name  "Physicsay"

git config user.email "physicsay@example.com"
git config user.name  "Physicsay"

git branch -M main
git push -u origin main
